`security-engineer` 서브에이전트를 사용해 토론배틀 프로젝트의 보안을 점검하세요.

## 점검 대상
$ARGUMENTS

(위가 비어 있으면 **전체 보안 감사**를 수행합니다: firestore.rules · Firebase Auth · API 키/시크릿 노출 · XSS·입력 검증 · Cloudflare Functions · AI 프롬프트 인젝션 · 의존성 취약점.)

## 수행 지시
1. `security-engineer` 에이전트에게 위 대상의 보안 감사를 위임한다.
2. 발견된 문제는 **심각도(심각/높음/보통/낮음)** 별로 정리하고, 각 항목마다 "무엇이 / 왜 위험한지 / 어떻게 고치는지"를 비개발자도 이해할 한국어로 설명한다.
3. firestore.rules에 고쳐야 할 부분이 있으면 **구체적인 수정안(diff)** 을 보여주고, 사용자가 "적용해줘"라고 하면 그때 반영한다. (그 외 소스 코드는 직접 고치지 말고 제안만 한다.)
4. 보고 마지막에 **"지금 당장 해야 할 일 1·2·3"** 을 우선순위 순으로 짧게 제시한다.

## 비개발자를 위한 보고 원칙
- 전문용어(RLS, CORS, XSS 등)는 괄호로 쉽게 풀어 쓴다.
- "남의 토론 점수를 조작할 수 있어요", "AI 키가 새면 요금 폭탄이 날 수 있어요"처럼 **실제로 무슨 피해가 생기는지** 와닿게 설명한다.
- 안전한 부분은 "여기는 괜찮아요"라고 분명히 알려 안심시킨다.
