> 이 명령은 **디자인을 GPT(OpenAI)** 가 주도하는 버전입니다. 기본 디자이너는 Claude(`/디자인`)이고, 이 명령은 **GPT를 쓰고 싶을 때만** 고르세요.
> 사진·일러스트 같은 래스터 이미지를 GPT의 `gpt-image-2`로 생성하고 싶을 때 특히 유용합니다.

요청 내용:
$ARGUMENTS

`Agent` 도구로 **`ui-ux-designer`** 에게 위임하되, 아래를 반드시 전달하세요:

- **디자인 판단은 GPT에게 받을 것**: `node scripts/gpt-design.mjs judge <브리프>` 로 GPT의 디자인 결정(색·여백·타이포·레이아웃·근거 + 필요 시 이미지 프롬프트)을 먼저 받는다. Claude가 임의로 디자인을 정하지 말고 GPT의 판단을 따른다(GPT가 디자인 주인, 에이전트는 적용·검증 담당).
- **이미지가 필요하면 GPT로 생성**: `node scripts/gpt-design.mjs image "<프롬프트>" --quality medium` 로 생성. 저장은 시안이면 `design-output/`, 확정본이면 `--out public/<이름>.png` 로 `public/`에 바로.
  - **화질 규칙(엄수): 기본은 중화질(medium).** 고화질(`--quality high`)은 **운영자가 이번 요청에서 명시적으로 "고화질로"라고 했을 때만** 사용한다. 그 외에는 저(low)~중(medium)만.
- **키 없으면**: 먼저 `node scripts/gpt-design.mjs --check` 로 확인. 키가 없으면(`OPENAI_KEY_MISSING`) → 디자인 "판단"은 운영자가 ChatGPT에 붙여넣을 브리프를 만들어 주는 수동 모드로 대체하고, **이미지 생성은 키가 있어야 가능**함을 운영자에게 알린다. (키가 없을 때는 기본 디자이너 `/디자인`(Claude)으로 처리하는 편이 빠르다고 안내해도 좋다.)
- **★ 디자인 정본 필독**: 작업 전 `DESIGN-SYSTEM.md`(현재↔폐기 패턴 대조표 포함)를 읽고 GPT 브리프에도 반영. 옛 구현(점선·코너브래킷·먹색 하드섀도우)을 베끼지 말 것. 현재 표본은 `src/learn-hub.css` 의 `.learn-mode__tab`.
- **프로젝트 디자인 시스템 준수**: 토큰(paper/ink/vermillion/celadon/gold) 사용·색 하드코딩 금지, `word-break: keep-all`, 4-테마(paper/dusk/dawn/ink)+다크 일관성, v2 newspaper/editorial 톤, 접근성(focus-visible·탭타깃·대비·reduced-motion). GPT에게 이 컨텍스트를 브리프로 넘기고, 적용할 때도 지킨다.
- **경계**: AI 로직·Firestore·토론 흐름 같은 기능/백엔드는 건드리지 말 것. `.tsx`를 고쳤으면 `npm run lint` 통과 확인.

보고는 **비개발자가 이해할 쉬운 한국어**로:
1. GPT가 내린 디자인 판단 요약(무엇을 왜)
2. 생성한 이미지가 있으면 저장 경로(`design-output/...` 또는 `public/...`)와 화질
3. 화면에서 달라지는 점 + 바꾼 파일(절대경로)
4. 확인 방법(`npm run dev` 후 어느 화면) · 다크/4-테마/모바일 메모
5. 다음 행동(예: "이 시안이 마음에 들면 public으로 확정할까요?")
