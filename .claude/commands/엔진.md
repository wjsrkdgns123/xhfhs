다음 작업을 **ai-debate-engineer** 서브에이전트에게 위임하세요. 이 에이전트는 AI 토론 엔진(`functions/api/ai/*`, `functions/_shared/claude.ts`)과 Firestore 데이터 모델/쿼리/집계의 1차 소유자입니다.

요청 내용:

$ARGUMENTS

처리 지침:
- 먼저 관련 파일(프롬프트 엔드포인트, App.tsx의 verdict/phase 블록, types.ts, 관련 hook)을 읽고 영향 범위를 파악할 것.
- 출력 계약(`<verdict>pro|con|tie</verdict>` 태그, 자수·라벨·줄 수)과 50/50 합산, phase 순서, 자동 연장(반드시 `pro_arg` 재시작)을 깨지 말 것.
- Haiku 호출 비용을 의식해 max_tokens·호출 단계를 불필요하게 늘리지 말 것.
- 보안 규칙(firestore.rules) 판단이 필요하면 직접 결론 내리지 말고 "/보안점검 필요"라고 지적만, 순수 화면/스타일/i18n은 "/개발"로 넘길 것.
- `.ts`/`.tsx`를 수정했으면 `npm run lint`(tsc 두 번)를 돌려 통과를 확인할 것.
- 비개발자도 이해할 쉬운 한국어로, 무엇을/왜/바꾼 파일/검증 결과/남은 것을 보고할 것.
