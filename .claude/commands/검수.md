토론배틀 프로젝트의 **QA 검수자(qa-engineer)** 서브에이전트에게 아래 검수 작업을 맡기세요.

## 검수 대상
$ARGUMENTS

(비어 있으면 현재 변경된 파일 전체를 대상으로 검수)

## qa-engineer 에게 지시할 것
1. 먼저 `npm run lint`(앱 tsc + Functions tsc 두 개)를 실제로 돌려 타입·회귀 오류를 확인.
2. 토론 흐름(phase 전환 / 자동 연장 / 50:50 판정 파싱) 회귀 점검.
3. i18n(KO/EN) 키 누락·미번역 점검.
4. 반응형·다크모드·4테마(paper/dusk/dawn/ink) 시각 점검 항목 산출.
5. 배포 전 체크리스트 출력 (main 푸시 = Cloudflare 자동 배포임을 명시).

## 보고 방식 (중요 — 운영자는 비개발자)
- 맨 위에 **"배포해도 됨 / 고쳐야 함"** 을 한 줄로 먼저 말할 것.
- 발견한 문제는 **무엇이·왜·어떻게 재현되는지**를 쉬운 한국어로 설명하고, 의심 파일 경로를 적을 것.
- 사람이 눈으로 봐야 하는 항목은 **체크박스 목록**으로 넘기고, 직접 확인 못 한 것은 "미검증"으로 표시.
- 코드를 직접 고치지는 말 것. 수정이 필요하면 "개발 담당에게 넘기세요"라고 안내.
