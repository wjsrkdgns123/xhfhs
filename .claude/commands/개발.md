토론배틀 프론트엔드 개발자(`frontend-developer` 서브에이전트)에게 아래 작업을 맡기세요.

## 요청 내용
$ARGUMENTS

## 개발자에게 지시할 것
1. 먼저 관련된 기존 파일(컴포넌트·훅·i18n)을 찾아 같은 패턴으로 구현하세요. 새로 만들기보다 재사용을 우선합니다.
2. 사용자에게 보이는 글자는 한국어(KO)와 영어(EN) 양쪽 i18n에 모두 넣으세요.
3. 색·폰트는 프로젝트 디자인 토큰만 쓰고, 다크 모드와 4테마(paper/dusk/dawn/ink)에서 깨지지 않게 하세요.
4. 코드를 고친 뒤에는 반드시 `npm run lint`를 돌려 오류 0을 확인하세요. 통과하기 전에는 끝났다고 하지 마세요.

## 결과 보고 방식 (비개발자용)
- 무엇을 바꿨는지 한 줄로 먼저 알려주세요.
- 고친 파일 목록과 각 파일이 무슨 역할인지 쉬운 말로 설명하세요.
- `npm run lint`가 통과했는지, 영어 번역도 같이 넣었는지 알려주세요.
- 브라우저에서 어디를 보면 결과를 확인할 수 있는지(예: 첫 화면, 토론방 `?room=...`) 안내하세요.
- 전문용어는 최대한 풀어서 설명하세요.
