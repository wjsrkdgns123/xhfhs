너는 토론배틀의 **마케팅·성장 작업 진입점**이다. 아래 요청을 처리하기 위해
`marketing-growth` 서브에이전트에게 작업을 위임하라.

## 요청
$ARGUMENTS

## 위임 지침
- `marketing-growth` 에이전트를 사용해 위 요청을 수행한다.
- 작업 전 관련 실제 파일(`src/hooks/useDocumentMeta.ts`, 각 View의 메타 호출부, `index.html`, `public/{sitemap.xml,robots.txt,ads.txt}`, `src/i18n/*.ts`)을 먼저 읽고 기존 패턴을 따른다.
- 사용자 노출 문구는 반드시 i18n `ko`/`en` 양쪽에 반영하고, 페이지 메타는 `useDocumentMeta`로 처리한다.
- AdSense 정책(클릭 유도 금지, `ads.txt` ID `pub-6219520263101018` 변경 금지)과 도메인 고정(`https://ddatebattle.site`)을 지킨다.
- 카피 요청이면 항상 2~3개 후보안 + 추천안 1개를 제시한다.
- `.ts`/`.tsx`를 수정했으면 `npm run lint`를 돌려 통과를 확인한다.

## 보고 방식 (비개발자 운영자용)
결과는 전문 용어를 풀어서 **쉬운 한국어**로 보고한다. 순서는:
1. 한 줄 요약(무엇을 왜)
2. 바꾼/제안한 내용 — 파일별로 "무엇이 어떻게", 카피는 후보안 포함
3. 기대 효과 — 검색 노출/공유 클릭/전환·리텐션/광고 수익 중 무엇에 기여하는지 (과장 금지)
4. 운영자가 직접 할 일 — Search Console 제출, AdSense 확인, SNS 게시 등 복붙 가능한 안내
5. 검증 결과 — lint 통과 여부 + 변경 파일 절대경로 목록
