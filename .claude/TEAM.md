# 토론배틀 AI 팀 사용설명서

> 토론배틀 프로젝트를 **역할별 전문 AI**에게 나눠 맡길 수 있게 만든 환경입니다.
> 채팅창에 `/명령` 을 입력하면 해당 분야 전문가가 작업하고, **비개발자도 이해할 수 있는 쉬운 한국어**로 보고합니다.

---

## 🚦 제일 쉽게 시작하기 — 그냥 `/팀`

무슨 일을 **누구에게 시켜야 할지 모르겠으면** 그냥 이렇게 하세요:

- `/팀` ← 입력하면: 전체 역할 목록과 "이럴 땐 이거 쓰세요"를 보여줍니다.
- `/팀 로비 화면이 모바일에서 깨져요` ← 평소 말투로 적으면: 알아서 **알맞은 담당자(여기선 디자이너)에게** 맡깁니다.

명령 이름을 외울 필요가 없습니다. 헷갈리면 항상 `/팀`.

---

## 👥 전문가 8명 (직접 부르기)

### ✏️ 만들기
| 명령 | 무슨 일을 하나 | 이럴 때 쓰세요 |
|---|---|---|
| `/기획 <아이디어>` | 새 기능을 기획 문서(무엇을·왜·언제 만들지)로 정리. **코드는 안 건드림** | "토론 다시보기 기능 어때?" "다음에 뭘 만들까?" |
| `/디자인 <요청>` | 화면의 색·여백·폰트·레이아웃·버튼 + **이미지**. **기본은 Claude가 직접 판단하고 그 자리에서 적용**(모바일·다크·테마·접근성 포함). GPT에게 맡기려면 `/디자인지피티` | "로비 카드 신문 스타일로" "OG 공유 카드 새로 그려줘" "모바일에서 헤더 깨져요" |
| `/디자인지피티 <요청>` | 위와 같지만 **디자인 판단·이미지 생성을 GPT(OpenAI)** 가 맡는 버전. 사진·일러스트를 `gpt-image-2`로 뽑고 싶을 때. `OPENAI_API_KEY` 필요 | "히어로 배경 일러스트 만들어줘" "OG 이미지 GPT로 뽑아줘" |
| `/개발 <요청>` | 화면·기능을 실제로 만들거나 고침 (실시간 연동·AI 연동·영어 번역까지, 검사 자동) | "로비 빈 상태에 안내 문구 추가" "방 목록 실시간 갱신 고쳐줘" |
| `/엔진 <요청>` | **AI 토론 엔진** — AI 사회자/토론자 말투, 승패 판정(verdict), 라운드 진행·자동연장, 챔피언·전적 통계 | "AI 사회자 말투 다듬어줘" "판정이 이상해" "챔피언 숫자가 안 맞아" |
| `/콘텐츠작성 <요청>` | 자료실·논리오류·용어사전·명토론·토론주제 글쓰기 + 한국어/영어 번역 | "논리 오류 3개 더 추가" "용어사전에 '입증책임' 넣어줘" |

> 🎨 **디자인은 기본이 Claude, 필요할 때만 GPT**: `/디자인`은 **Claude가 직접** 색·여백·레이아웃을 판단하고 그 자리에서 코드까지 적용합니다(외부 키 불필요). 아이콘·도장·OG 카드 같은 이미지는 신문/잉크 톤에 맞춰 **SVG로 직접** 그립니다. 사진·일러스트를 GPT의 **이미지(gpt-image-2)** 로 뽑고 싶으면 **`/디자인지피티`** 를 쓰세요(`.env`의 `OPENAI_API_KEY` 필요, 화질 **기본 중화질**·고화질은 "고화질로"라고 말할 때만). 시안은 `design-output/`에, 확정본은 `public/`에 저장됩니다.

### 🔎 점검
| 명령 | 무슨 일을 하나 | 이럴 때 쓰세요 |
|---|---|---|
| `/검수 <대상>` | **품질** 점검 — 코드 오류·토론 흐름·번역 누락·테마를 한 번에 검사하고 배포 전 체크리스트 제공 | "방금 바꾼 거 문제없어?" "배포 전 전체 점검" |
| `/보안점검 <대상>` | **보안** 점검 — DB 규칙·API 키 노출·해킹(프롬프트 인젝션)·취약점 감사 (비우면 전체) | "이거 안전해?" "firestore 규칙 봐줘" |

> 💡 `/검수`(품질 = 안 깨졌나) 와 `/보안점검`(보안 = 안 털리나)은 **다른 일**입니다. 헷갈리면 둘 다 한 번에 하는 `/배포준비`를 쓰세요.

### 📣 키우기
| 명령 | 무슨 일을 하나 | 이럴 때 쓰세요 |
|---|---|---|
| `/마케팅 <요청>` | 검색 노출(SEO)·광고 수익(AdSense)·공유 카드·랜딩 문구·유입/재방문 전략 | "토론방 SEO 보강" "히어로 문구 3개 만들어줘" "유입 전략" |

## 🤝 묶음 명령 3개 (여러 전문가가 협업 — 가장 편한 버튼)

| 명령 | 무슨 일을 하나 |
|---|---|
| `/팀 [할 말]` | **안내 데스크 겸 팀장.** 비우면 전체 역할 안내. 요청을 적으면 — **한 사람 일이면** 그 담당에게, **여러 사람 일이면** 역할별로 작업을 나눠 **동시에(병렬)** 일하고 → 합쳐서 → **검수·테스트**한 뒤 보고. 한 명이 다 떠안지 않음 |
| `/새기능 <아이디어>` | 막연한 아이디어 한 줄 → **기획 → (승인 확인) → 디자인·구현 → 검수**까지 자동 진행. **승인 전엔 코드 안 건드림** |
| `/배포준비 [범위]` | "올려도 돼?" → **품질 검수 → 보안 점검**을 순서대로 돌려 "배포해도 됩니다 / 보류"를 판정 (직접 푸시는 안 함) |

---

## ⌨️ 쓰는 법

1. 채팅창에 `/` 를 치면 명령 목록이 뜹니다.
2. 명령을 고르고, **뒤에 평소 말투로 하고 싶은 말**을 적으면 됩니다.
   - 예) `/디자인 로비 방 카드를 신문 기사 스타일로 바꿔줘`
   - 예) `/새기능 토론 끝나면 명장면을 카드로 공유하는 기능`
3. 전문용어를 몰라도 됩니다. **그냥 한국어로 평소처럼** 말하세요.

## 🔄 자동으로 되는 것 (자동화)

- **알아서 맡김**: 꼭 명령을 안 써도, "로비 색이 칙칙해"처럼 평소처럼 부탁하면 Claude가 알맞은 전문가(디자이너)에게 자동으로 넘겨서 처리합니다.
- **나눠서 동시에(병렬) 협업**: `/팀`에 **여러 손이 필요한 일**을 적으면, 한 명이 다 떠안지 않고 역할별로 **작업 구역을 나눠 동시에** 일한 뒤 → **합쳐서 → 검수·테스트** → 한 번에 보고합니다. (예: 디자이너가 화면을 짜는 동안 콘텐츠 작가가 글을 쓰고, 둘이 모이면 개발자가 붙이고, 마지막에 검수)
- **순서대로 협업**: `/새기능`·`/배포준비`는 여러 전문가를 **자동으로 순서대로** 불러서 한 번에 끝냅니다.
- **검사 자동**: 코드를 고치는 전문가(`/개발`·`/엔진`·`/콘텐츠작성` 등)는 작업 후 **스스로 오류 검사(lint)** 를 돌리고, 통과해야 "끝났다"고 보고합니다.

## 🛠️ 더 바꾸고 싶으면

- 역할을 **추가/수정**하고 싶으면 그냥 말하세요. 예) "마케터한테 방문자 통계 분석도 맡기고 싶어", "배포까지 자동으로 해주는 역할 만들어줘".
- 전문가 정의 파일: `.claude/agents/*.md` · 명령 파일: `.claude/commands/*.md` (직접 안 건드려도 됩니다 — 말로 부탁하면 제가 고쳐드립니다.)
- 이 팀 구성은 토론배틀 코드(React·Firestore·Cloudflare·AI 엔드포인트·9개 i18n·4테마)에 맞춰 설계됐습니다.

## 📌 한눈 요약

```
모르겠다           →  /팀
새 기능 만들고 싶다  →  /새기능   (기획부터 검수까지 한 번에)
올리기 전 점검      →  /배포준비  (품질 + 보안)

화면 디자인         →  /디자인        (기본 Claude / GPT로 하려면 /디자인지피티)
기능 구현·수정      →  /개발
AI·판정·통계        →  /엔진
교육 콘텐츠 글쓰기   →  /콘텐츠작성
기획·우선순위       →  /기획
마케팅·SEO·광고     →  /마케팅
품질 검사           →  /검수
보안 검사           →  /보안점검
```
